Internet Explorer, Attribute Selectors, and Really Long Attribute Values

Jun 21st, 2013 | Comments

In Internet Explorer, attribute selectors that inspect the attribute’s value do not work if the value is >= 4096 characters. This is one of those bugs that you are highly unlikely to run into, but is incredibly painful to track down if you do.

As an example, take the following:

<style>
    /* Elements that have the attribute */
    div[data-test] { background: red; }

    /* Elements that have the attribute, and its value contains "Nullam" */
    div[data-test*="Nullam"] { color: blue; }

    /* Elements that have the attribute, and its value starts with "Lorem" */
    div[data-test^="Lorem"] { border: 1px solid black; }
</style>

The example has two <div>s and 3 rules applied to both elements via attribute selectors. In Firefox, Chrome, Safari, and Opera all rules apply as they should and the elements display as follows:

In Internet Explorer however, the display looks like this:

The selector that looks for the existence of the attribute, div[data-test], matches both elements and the background: red rule is applied.

However, the two selectors that inspect the attribute’s values, div[data-test*="Nullam"] and div[data-test^="Lorem"]), mysteriously do not match the <div> with the 4096 character data-test attribute.
