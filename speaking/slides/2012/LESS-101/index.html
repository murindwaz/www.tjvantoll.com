LESS 101

CSS Preprocessor

  • Write LESS, generate cross browser friendly CSS.
  • Extension of CSS, therefore any valid CSS is valid LESS.
  • Written in JavaScript - Runs in the browser or via Node.js

Client Side Usage

Development:

<link rel="stylesheet/less" href="css/styles.less">
<script src="less.js"></script>

Production:

<link rel="stylesheet" href="css/styles.css">

Watch Mode

Watch for .less file changes client-side re-paint the page automatically.

To activate it you have 2 options:

  • Append #!watch to the URL.
  • Run less.watch() from the console.

Command Line

Default functionality:

$ lessc style.less

Choose the output location:

$ lessc style.less > myFile.css

Also minify the CSS output:

$ lessc -x style.less > style.css

So what can you do with LESS?

Variables!!!!

LESS:

@my_favorite_color: #abc123;

body {
    color: @my_favorite_color;
}

CSS:

body {
    color: #abc123;
}

Note that variables in LESS are actually constants; they cannot be redefined.

Comments

Multi line comments of single line comments can be used.

/* Multi
   Line
   Comment */
body { color: red; } //Single line comment

Multi line comments are preserved in the outputted CSS, single line ones are removed.

If you minify the CSS all comments are wiped.

Nesting

With LESS you can nest rules instead writing out full rules every time. Therefore the following CSS... 

#header { color: red; }
#header p { font-size: 1.2em; }
#header p span { color: blue; }

Could be rewritten as...

#header {
    color: red;
    p {
        font-size: 1.2em;
        span {
            color: blue;
        }
    }
}

Fancier Nesting

You can use the & operator to have nested selectors concatenated to their parent's selector (instead of acting as a descendant).

CSS

a { color: blue; }
a:hover { color: red; }
a.noHover { color: blue; }

LESS

a {
    color: blue;
    &:hover { color: red; }
    &.noHover { color: blue; }
}

Math

Any number, color, or variable can be operated on.

@default_border_width: 1px;

.container {
    border: (@default_border_width + 2) solid black;
}

CSS

.container {
    border: 3px solid black;
}

Math on Colors

@base_color: #222;
.container {
    color: @base_color * 4;
    background-color: @base_color + '#111';
}

CSS

.container {
    color: #888888;
    background-color: #333333;
}

More Color Stuff

LESS provides a number of useful functions to transform colors.

lighten(@color, 10%);     // return a color which is 10% *lighter* than @color
darken(@color, 10%);      // return a color which is 10% *darker* than @color

saturate(@color, 10%);    // return a color 10% *more* saturated than @color
desaturate(@color, 10%);  // return a color 10% *less* saturated than @color

fadein(@color, 10%);      // return a color 10% *less* transparent than @color
fadeout(@color, 10%);     // return a color 10% *more* transparent than @color
fade(@color, 50%);        // return @color with 50% transparency

spin(@color, 10);         // return a color with a 10 degree larger in hue than @color
spin(@color, -10);        // return a color with a 10 degree smaller hue than @color

mix(@color1, @color2);    // return a mix of @color1 and @color2

Mixins

Include the properties from one ruleset in another.

.colorful {
    color: green;
    background-color: hotpink;
    border: 6px solid skyblue;
}

.colorful_spaced {
    .colorful;
    padding: 20px;
}

Any class or id ruleset can be mixed-in as such.

Parametric Mixins

Mixins can take parmeters for configuration.

.color_maker(@color) {
    color: @color;
    background-color: @color;
}

#my_element {
    .color_maker(yellow);
}

CSS

#my_element {
    color: yellow;
    background-color: yellow;
}

Default Parameters for Mixins

Mixins parameters can optionally have default vales.

.has_margin(@margin: 5px) {
    margin: @margin;
}

h4 { .has_margin; }
h5 { .has_margin(3px); }

CSS

h4 { margin: 5px; }
h5 { margin: 3px; }

Importing

You can import .less files and all variables and mixins will be available in the main file. The .less extension is optional so both of these work: 

@import 'myFile.less';
@import 'myFile';

You can also import CSS files by the same mechanism.

@import 'myFile.css';

String Interpolation

Variables can be embedded inside strings using the @{name} construct.

@base_image_url: 'http://www.liquidweb.com/images/';

.logo {
    background-image: url('@{base_image_url}/logo.png');
}
